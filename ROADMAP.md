# Zoxide Doctor Roadmap

## Current Status

v0.2.0 is tagged and published (release archives + SHA256SUMS via GitHub Actions, generated Homebrew formula). Remaining work: Homebrew tap plus the upstream to-dos below.

Locked decisions:

- Primary workflow: `z <query>` first, then no-arg `zdr` if zoxide lands in the wrong directory.
- Repeated no-arg `zdr` calls are the rejection/escalation signal.
- `zdr <query>` is an experimental direct lookup, not the default zoxide replacement path.
- Recovery context comes from recorded shell state, not shell history scraping.
- Correction memory is zdr-owned state, not zoxide frecency boosting.
- Runtime target: TypeScript on Bun, distributed as a Bun-compiled executable.
- Provider layer: Pi provider/model SDK (`@earendil-works/pi-ai`); completion/model APIs only — no Pi agent loop, harness, TUI, tool execution, or session machinery.
- Default provider: `openai-codex` / `gpt-5.6-terra` at minimal reasoning effort, authenticated via ChatGPT-plan OAuth (`zdr provider-login openai-codex`). Env-key providers (e.g. `OPENROUTER_API_KEY`) are the documented alternative.
- Cold start: 150ms local budget (`scripts/timing-baseline.ts`); provider SDK imports stay dynamic.

## Shipped

### v0.1.0 — Core Recovery Loop

Shell wrappers for zsh, bash, and fish record `z` attempts; no-arg `zdr` repairs the last bad jump from zoxide candidates with a bounded, redacted prompt and a strict JSON `candidate_id` response. Second attempt escalates, third opens an `fd` + `fzf` picker. `zdr <query>` direct lookup with `corrections.json` memory (cache-first, stale-path eviction). Local telemetry (tokens, cost, prompt-cache fields where exposed) with `zdr debug-events` / `zdr prune-events`, timing diagnostics, v1 config file, install docs.

### v0.2.0 — Providers, Escalation Tier, Evals

- `openai-codex` / `gpt-5.6-terra` default via ChatGPT-plan OAuth; measured 96% eval accuracy, 100% stability across 3x50 repeats, p50 ~1.4s.
- Optional escalation tier for the second no-arg `zdr` (`zdr config-escalation <backend> <model>`, recommended `claude sonnet`); Claude backend runs on `@anthropic-ai/claude-agent-sdk` using the local `claude` login.
- Read-only import of Pi CLI OAuth logins (`~/.pi/agent/auth.json`) into zdr's own store.
- `zdr provider-discover`: token-free per-backend readiness report with tier summary.
- Alias memory: recovery-mode picker selections and high-confidence repairs stored in `corrections.json` and injected as the top candidate on later attempts.
- Eval suite (`src/eval/`, 50 cases, 10 categories): offline recall via `bun scripts/run-evals.ts`, spend-gated live runs, telemetry-to-case mining.
- Codex client-identity shim (`src/provider/codex-identity.ts`) so `gpt-5.6-luna` routes correctly.
- Release automation: archives, SHA256SUMS, Homebrew formula generated by `bun run release:prepare`.
